# PDF 转 Markdown 提示词

你需要将一本教材 PDF 的指定章节提取并整理为 Markdown 文件。请严格按照以下要求执行：

## 任务目标

将教材中的指定章节提取为 `.md` 文件，内容应尽量忠实于书中原文，排版清晰，章节层级规范，数学公式使用可正常渲染的 LaTeX 重写。

## 处理流程

1. 先检查 PDF 是否为扫描版。
   - 如果 PDF 可直接提取文本，优先使用文本提取。
   - 如果 PDF 是扫描版，需要先渲染页面，再进行 OCR。

2. 定位章节起止页。
   - 通过目录页、章节标题页和下一章标题确认边界。
   - 只提取目标章节，不要混入下一章内容。

3. 生成 Markdown。
   - 按书中结构整理标题层级。
   - 合并 OCR 造成的断行。
   - 修正常见 OCR 错字、乱码和错误空格。
   - 保留必要的页码来源说明，可放在文件开头注释中。

4. 公式处理。
   - 公式必须使用 LaTeX 重写。
   - 不要直接保留 OCR 识别出的散乱公式文本。
   - 矩阵、向量、求和、积分、迹、行列式、转置、逆、特征值等公式都应整理成标准 LaTeX。
   - 坐标系、坐标轴和角度变量中的下标必须按原 PDF 核对并用 LaTeX 表示，不要把下标误识别成逗号、句点、普通数字或正文字符。例如：`O_E-X_IY_IZ_I`、`O_E-X_EY_EZ_E`、`O_EX_E`、`\phi_0`、`B_0`、`\alpha_0`、`A_0`。若 OCR 出现 `O-X,Y,Z,`、`Oe`、`O.Z`、`B。`、`A.`、`0-xyz` 等形式，应回看原图修正。

## LaTeX 渲染兼容规则

当前 Markdown 环境可解析 `$$ ... $$` 公式块，并使用 KaTeX 渲染。请遵守以下规则：

1. 使用块公式：

```latex
$$
...
$$
```

2. 矩阵使用 KaTeX 支持的 `bmatrix`：

```latex
$$
\begin{bmatrix}
1 & 2 & 3\\
3 & 2 & 1
\end{bmatrix}
$$
```

3. 不要使用老式 TeX `\matrix{...}`，KaTeX 会报错：

```latex
\matrix{1 & 2\\ 3 & 4}
```

4. 带编号的公式外层仍使用 Markdown 的 `$$...$$` 公式块，内部使用 `equation` 环境，并用 `\tag{...}` 写入教材原编号：

```latex
$$
\begin{equation}
p=\frac{\bar R}{\mu}\rho T
\tag{1-2-1}
\end{equation}
$$
```

5. 不要让 `=`、`+`、`-` 等操作符在公式块中单独占一行。某些 Markdown 渲染器会在数学渲染前先应用 Markdown 语法，把单独一行的 `=` 当作 Setext 标题下划线，导致公式块解析失败、等号消失、上一行被放大成标题。

错误示例：

```latex
$$
\begin{equation}
P(A_i\mid B)
=
\frac{P(B\mid A_i)P(A_i)}{\sum_j P(B\mid A_j)P(A_j)}.
\tag{2.9}
\end{equation}
$$
```

推荐写法：

```latex
$$
\begin{equation}
P(A_i\mid B)=\frac{P(B\mid A_i)P(A_i)}{\sum_j P(B\mid A_j)P(A_j)}.
\tag{2.9}
\end{equation}
$$
```

完整示例：

```latex
$$
\begin{equation}
\begin{bmatrix}
1 & 2 & 3\\
3 & 2 & 1
\end{bmatrix}
+
\begin{bmatrix}
0 & 4 & 1\\
1 & -1 & -2
\end{bmatrix}
=
\begin{bmatrix}
1 & 6 & 4\\
4 & 1 & -1
\end{bmatrix}.
\tag{1.12}
\end{equation}
$$
```

6. 尽量避免使用渲染器可能不支持的复杂宏。若必须使用，优先选择 KaTeX 支持的写法。

7. 常用符号写法：

```latex
\operatorname{Tr}(A)
\det(A)
\sum_{i=1}^{n}
\int_{0}^{T}
A^{-1}
A^T
A^H
\lambda_i
\rho(A)
```

## Markdown 格式要求

1. 章节标题示例：

```markdown
# 第 1 章 线性系统理论

## 1.1 矩阵代数和矩阵运算

### 1.1.1 矩阵代数
```

2. 正文段落应自然合并，不要保留 OCR 造成的每行断句。

3. 公式前后保留空行，便于 Markdown 渲染。

4. 习题部分可保留为编号列表。

5. 不要输出乱码，不要保留明显 OCR 残片。

## 输出要求

最终输出一个 Markdown 文件，文件名应清晰表达章节内容，例如：

```text
第一章_线性系统理论.md
```

完成后需要检查：

- 目标章节是否完整。
- 是否混入下一章。
- 是否仍有 OCR 乱码。
- `$$` 公式块是否成对。
- 是否存在 `\matrix{...}`。
- 带编号公式是否外层使用 `$$...$$`，内部使用 `equation` 环境，并把编号写成 `\tag{...}`。
- 数学公式块中是否存在单独占一行的 `=`、`+`、`-`。
- 矩阵公式是否能在 KaTeX 环境下正常渲染。

## 补充规范

### 术语解释

如果需要对正文中的术语、概念或背景知识进行解释，不要使用脚注形式，因为脚注跳转会打断阅读。优先在相关段落后方就近插入 Markdown/HTML 折叠块：

```markdown
<details>
<summary>术语名</summary>

这里写解释内容。解释应简洁、贴近上下文，不改变原文主线。

</details>
```

折叠块应默认收起，标题简短，例如“控制力”“力矩”。这样读者可以连续阅读主文，也可以按需展开补充说明。

### 不要保留分页

将 PDF 转为 Markdown 时，不要在正文中插入 `<!-- PDF page ... -->`、页码、分页线或其他会让渲染后阅读不连续的分页标记。可以在文件开头的 HTML 注释中说明来源页码范围，但正文应作为连续文章呈现。

转换后要检查并修复 PDF 分页造成的断句、断词和硬换行，例如：

```text
人为地以一定高度划出一条边界作为大气边界
层。
```

应合并为：

```text
人为地以一定高度划出一条边界作为大气边界层。
```

类似“原/因”“物/体”“火箭整/体”等跨页或跨行断开的词，也应在整理 Markdown 时合并。

### 保留原 PDF 的自然段落

转换教材 PDF 时，应按原 PDF 的自然段落划分 Markdown 段落：同一自然段内的 OCR 行应合并为连续文字，段落之间用一个空行分隔。不要为了贴合 PDF 版面而逐行硬换行，也不要把多个自然段合并成一个超长段落。

处理原则：

- 标题、图题、公式块前后保留空行。
- 正文同一自然段内合并 OCR 行，保持文字内容完整。
- 不保留 PDF 分页标记。
- 跨页或跨行造成的断词、断句需要合并，例如“大气边界\n层”应合并为“大气边界层”。
- 判断自然段时优先参考原 PDF 的段首缩进、标题层级、编号段落和上下文语义。
### 区分正文图引用和图题

整理 OCR 文本时，要区分正文中的图引用和真正的图题。正文里的“如图1-2所示”“见图1-6”等属于句子的一部分，应合并回原段落，不要改成斜体图题，也不要单独成段。

错误示例：

```markdown
两者总称大地水准面，如

*图1-2所示。大地水准面的表面是连续的、封闭的，而且没有皱褶与裂痕。故是*

一个等重力势面。
```

正确处理：

```markdown
两者总称大地水准面，如图1-2所示。大地水准面的表面是连续的、封闭的，而且没有皱褶与裂痕，故是一个等重力势面。
```

只有独立出现并完整描述图片标题的内容，例如“图1-2 地球物理表面、大地水准面与地球椭球体示意图”，才应作为图题处理。

### 不要把图内标注当作正文

OCR 常会把图片内部的文字标注识别成普通正文，例如坐标轴名、箭头标注、图中对象名称等。整理 Markdown 时，这类图内标注不应作为正文段落渲染出来；应删除这些散落的图内文字，只保留真正的图题，或在需要时用图片/图示说明承载。

判断方法：如果一组短文本出现在图题前后，内容像“地球质量中心”“大地水准面”“p”“p′”“x/y/z 轴”等，且无法连成正文句子，通常就是图内标注，应从正文中移除。

### 公式编号使用 equation tag

教材中的公式编号（如 `(1-2-1)`）不要单独作为正文或单独公式块输出，也不要写成 `\qquad (1-2-1)`。带编号公式应外层使用 Markdown 的 `$$...$$` 公式块，内部使用 `equation` 环境，并把教材编号写成该公式的 `\tag{...}`。公式本体必须回看原 PDF 或截图补全，不能只留下编号。

推荐写法：

```latex
$$
\begin{equation}
p=\frac{\bar R}{\mu}\rho T
\tag{1-2-1}
\end{equation}
$$
```

错误写法：

```latex
$$
(1-2-1)
$$
```

也不要把编号散落在公式块外。若 OCR 只识别出编号而漏掉公式，应先补全公式，再用 `$$` 包裹的 `equation` 环境和 `\tag{...}` 输出。

转换完成后必须扫描两类问题：

- 是否仍有 `\qquad (1-2-1)` 这类把编号直接拼在公式末尾的写法；若有，应改成 `$$` + `equation` + `\tag{...}`。
- 是否存在只有编号的公式块，例如：

```latex
$$
(1-2-1)
$$
```

这说明 OCR 漏识了公式本体，不能把空公式强行改成 `\tag{...}`，必须回看原 PDF 或截图补全公式后再输出。

### 坐标系符号和下标

教材中的坐标系、坐标轴和方向余弦阵符号常含有下标，OCR 容易把下标识别成逗号、句号、普通字母或残缺文本。例如 `O_p-x_p y_p z_p` 可能被识别为 `o，-x,y,z,`，`x_q` 可能被识别为 `x,` 或 `x。`，`P_{pq}` 可能被识别为 `P。`、`P,` 或 `Po`。

整理时应根据上下文恢复为 Markdown LaTeX：

```markdown
设 $O_p-x_p y_p z_p$ 及 $O_q-x_q y_q z_q$ 为……
令 $P_{pq}$ 是把 $x_q$、$y_q$、$z_q$ 坐标轴单位矢量变换成 $x_p$、$y_p$、$z_p$ 坐标轴单位矢量的转换矩阵。
```

方向余弦阵、单位矢量列阵和元素下标也应写成 LaTeX，例如 `$E_p$`、`$E_q$`、`$P_{pq}$`、`$a_{ij}$`、`$x_p\cdot y_q$`。不要保留 `x,`、`z。`、`M,[]`、`E,`、`P。` 这类 OCR 残留。

当坐标系以大写字母命名时，转换矩阵下标也要完整保留两个坐标系标识。例如图1-7中 $Q$ 坐标系是小写 $o-xyz$，$P$ 坐标系是大写 $O-XYZ$，`$P_{PQ}$` 表示由 $Q(o-xyz)$ 到 $P(O-XYZ)$ 的方向余弦阵；不要简写成 `$P_Q$`，也不要把 `$P_{PQ}$` 误识别为 `P_Q`、`P。` 或正文中的普通 `P`、`Q`。

图示说明中若涉及两个坐标系，也必须显式写出坐标系标识和公共原点，避免只写“两坐标系”导致图内关系不可辨认。例如图题或图片 `alt` 可写成：

```markdown
图1-7 $Q(o-xyz)$ 经绕 $z$、$x_1$、$z_2$ 三次转动得到 $P(O-XYZ)$ 的欧拉角关系图（$Q$ 为小写坐标系，$P$ 为大写坐标系，原点重合，交点线为 $N$）
```

注意区分字母 `O/o` 与数字 `0`。若原文是坐标系原点，应写作 `$O$` 或 `$o$`；若是单位矢量上标，可写作 `$\boldsymbol{x}^0$`、`$\boldsymbol{y}^0$`、`$\boldsymbol{z}^0$`，不要误写成 `x°`、`x。` 或 `x,`。
